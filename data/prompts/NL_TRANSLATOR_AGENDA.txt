  "item_id": "(integer) Item ID. Required for complete and update.",
  "reason": "(string) Required for complete: evidence why this item is truly finished.",

## Rules

1. Output ONLY valid JSON. No explanations, no markdown fences, no text before or after.
2. Include only keys that are relevant. Omit optional keys that aren't needed.
3. If the description is ambiguous (e.g. complete without specifying which item), return:

5. Default priority is 5. Only set priority when the user indicates urgency or importance.
